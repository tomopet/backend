## 2. 백엔드 코드 및 협업 규칙 (Convention)

둘이서 관리하는 만큼 서로의 코드를 한눈에 알아볼 수 있도록 약속합니다.

### 네이밍 및 패키지 구조
- **패키지 구조:** 기능 단위(Domain) 중심 패키지 구성을 지향합니다.
  - *예시: `com.tomopet.backend.domain.user`, `com.tomopet.backend.domain.dog`*
- **클래스명:** 파스칼 케이스(PascalCase) 사용 및 역할 명시
  - *예시: `UserController`, `UserService`, `UserRepository`, `UserResponseDto`*
- **DB 테이블 및 컬럼명:** 스네이크 케이스(snake_case) 소문자 사용
  - *예시: `user_id`, `dog_profile_image`, `created_at`*

### API 응답 표준 포맷 (Response Wrapper)
프론트엔드가 에러나 성공 데이터를 일관되게 처리할 수 있도록 모든 API 응답은 아래 포맷으로 통일합니다.

```json
{
  "success": true,
  "message": "요청이 성공적으로 처리되었습니다.",
  "data": { ... } // 실패 시에는 null
}
